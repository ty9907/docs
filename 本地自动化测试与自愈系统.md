## 2. 核心交付件详细设计

### 2.1 项目元信息文件：`project.yaml`

定义 multica 项目全局设置，如默认 LLM、工具包路径、环境变量等。

```yaml
# project.yaml
name: auto-test-healer
version: 1.0.0
description: 本地自动化测试与自愈系统
default_llm:
  provider: openai
  model: gpt-4o
  temperature: 0.2
tool_packages:
  - tools.backend_tools
  - tools.frontend_tools
  - tools.test_tools
  - tools.common_utils
mcp_servers:
  jetbrains:
    config: mcp_configs/jetbrains_mcp.json
  playwright:
    config: mcp_configs/playwright_mcp.json
human_in_the_loop:
  enabled: true
  timeout: 300   # 秒
```

### 2.2 主工作流：`task/main.task.yaml`

控制整个测试-监控-修复循环，是多 Agent 协作的蓝图。

```yaml
# main.task.yaml
name: 全栈测试与自愈
entry_agent: orchestrator

steps:
  - parallel:
      - backend_agent.monitor_and_heal
      - frontend_agent.monitor_and_heal

  - wait_for_health:
      backend: "http://localhost:8080/actuator/health"
      frontend_keyword: "Compiled successfully"
      timeout: 60

  - test_agent.execute_cases:
      cases_file: "cases/e2e_cases.yaml"

  - loop:
      condition: "test_agent.failed_cases_count > 0 and retry_count < max_retries"
      steps:
        - orchestrator.dispatch_anomaly:
            anomaly: "{{ last_anomaly_event }}"

        - human_input:
            prompt: "是否应用以上修复？(accept/reject/modify)"
            var: user_decision

        - condition:
            - if: "user_decision == 'accept'"
              then:
                - apply_fix:
                    agent: "{{ fix_agent }}"
                    patch: "{{ fix_patch }}"
                - wait_for_reload:
                    backend_keyword: "Restarted"
                    frontend_keyword: "Compiled successfully"
                    timeout: 30
                - test_agent.retry_failed_and_downstream
            - if: "user_decision == 'reject'"
              then:
                - abort: "用户拒绝修复，流程终止"
            - if: "user_decision == 'modify'"
              then:
                - human_input:
                    prompt: "修改完成后输入 'continue'"
                - test_agent.retry_failed_and_downstream

  - test_agent.generate_report
```

**关键说明**：
- `orchestrator.dispatch_anomaly` 会根据异常来源（后端/前端/测试）自动调用相应 Agent 的分析工具。
- `human_input` 为 multica 内置交互步骤，执行时终端暂停并等待输入。
- `apply_fix` 是一个复合动作，内部根据 Agent 类型调用不同的工具。

### 2.4 工具（Skills）Python 实现范例

#### 2.4.1 `tools/common_utils.py`（通用工具）

```python
import asyncio
import httpx
import json
from pathlib import Path

def read_file(file_path: str) -> str:
    """读取文件内容，注册为multica Tool"""
    return Path(file_path).read_text(encoding="utf-8")

async def wait_for_health(url: str, timeout: int = 60) -> bool:
    """轮询健康检查端点"""
    async with httpx.AsyncClient() as client:
        start = asyncio.get_event_loop().time()
        while asyncio.get_event_loop().time() - start < timeout:
            try:
                resp = await client.get(url, timeout=5)
                if resp.status_code == 200:
                    return True
            except Exception:
                pass
            await asyncio.sleep(2)
    return False

def generate_diff(original: str, modified: str) -> str:
    """生成unified diff字符串（简化版，实际可用difflib）"""
    import difflib
    diff = difflib.unified_diff(
        original.splitlines(keepends=True),
        modified.splitlines(keepends=True),
        fromfile='original',
        tofile='modified'
    )
    return ''.join(diff)
```

#### 2.4.2 `tools/backend_tools.py`

```python
from tools.common_utils import read_file
from multica import tool, get_mcp_client

@tool
async def analyze_backend_exception(logs: str, source_file: str, line_number: int) -> dict:
    """
    分析后端异常，调用LLM生成修复建议。
    返回 dict: { patch: str, explanation: str }
    """
    code = read_file(source_file)
    prompt = f"""你是Java专家。根据以下异常日志和源代码，分析根本原因并生成修复补丁（unified diff格式）。
异常日志：{logs}
源代码文件：{source_file}
行号：{line_number}
代码内容：{code}
输出JSON格式，包含patch和explanation字段。"""
    # 此处调用LLM API（multica环境下可使用内置llm call）
    response = await llm_call(prompt)
    return json.loads(response)

@tool
async def apply_backend_patch(file_path: str, patch: str):
    """
    应用后端补丁。对于Java文件，直接写入新内容，依赖IDEA自动编译。
    """
    original = read_file(file_path)
    # 简单patch应用（使用difflib或调用系统patch命令）
    # 生产级代码应更健壮
    new_content = apply_unified_diff(original, patch)
    from pathlib import Path
    Path(file_path).write_text(new_content, encoding="utf-8")
```

#### 2.4.3 `tools/frontend_tools.py`

```python
import subprocess, asyncio
from multica import tool

@tool
async def start_dev_server(cmd: str, cwd: str):
    """启动前端dev server，返回异步生成器，逐行yield日志"""
    process = await asyncio.create_subprocess_shell(
        cmd,
        cwd=cwd,
        stdout=subprocess.PIPE,
        stderr=subprocess.STDOUT
    )
    # 返回一个可迭代流，由multica Agent在循环中消费
    return process

@tool
async def write_file(file_path: str, content: str):
    """写入前端源文件，触发HMR"""
    from pathlib import Path
    Path(file_path).write_text(content, encoding="utf-8")

@tool
async def analyze_frontend_exception(error_msg: str, file_path: str) -> dict:
    """类似后端，生成前端修复建议"""
    code = Path(file_path).read_text()
    prompt = f"根据前端错误信息生成修复补丁... (同前)"
    response = await llm_call(prompt)
    return json.loads(response)
```

#### 2.4.4 `tools/test_tools.py`

```python
import subprocess, base64, json
from multica import tool

@tool
async def execute_db_query(query: str, db_type: str = "mysql") -> str:
    """执行数据库查询并返回结果"""
    if db_type == "mysql":
        # 根据项目配置获取连接信息
        cmd = f"mysql -e \"{query}\""
    proc = await asyncio.create_subprocess_shell(cmd, stdout=subprocess.PIPE)
    stdout, _ = await proc.communicate()
    return stdout.decode()

@tool
async def multimodal_visual_check(screenshot_base64: str, expectation: str) -> dict:
    """
    调用多模态模型验证截图是否匹配预期描述
    返回 { passed: bool, reason: str }
    """
    prompt = f"""你是UI验证专家。请根据截图和预期描述判断页面样式是否符合要求。
预期描述：{expectation}
请输出JSON: {{"passed": bool, "reason": "..."}}"""
    # 调用支持视觉的LLM API，传入image参数
    response = await llm_vision_call(prompt, image_base64=screenshot_base64)
    return json.loads(response)
```

## 3. 打包与运行

### 3.1 打包方式
整个 `auto-test-healer/` 目录即为一个 multica 项目。可压缩为 `.zip` 或发布到内部 Git 仓库。

### 3.2 用户使用步骤
1. 将项目克隆到本地，修改 `config/project_template.yaml`（或重命名为 `project.yaml`）。
2. 安装 multica：`pip install multica`
3. 确保 IDEA 已安装 JetBrains MCP 插件且项目可运行。
4. 安装 Playwright：`npx playwright install`
5. 在 `auto-test-healer/` 目录下执行：
   ```bash
   multica run task/main.task.yaml
   ```
6. 终端将显示服务启动日志、测试执行过程。当出现异常修复建议时，等待用户输入 `accept`/`reject`/`modify`。
7. 所有通过或最终失败后，系统输出测试报告。

### 3.3 集成到 IDE
可配置 IDEA 外部工具，绑定快捷键直接调用 `multica run` 命令，实现一键启动。

## 4. 总结

本设计将完整的测试自愈系统分解为 **项目配置、工作流、Agent 定义、Python 工具集、MCP 连接、测试用例** 六大类交付件，所有组件均采用 multica 原生支持的 YAML + Python + MCP 标准构建，无额外依赖。  
通过清晰的分离关注点，开发团队可以快速根据自身项目调整配置和用例，复用全部 Agent 协作逻辑，实现从“写完代码”到“自动化验证修复”的本地开发体验闭环。
